# chanlun.c99 — 缠论技术分析库（Python 参考实现 + C99 移植）

提供 **Python** 与 **C99** 双语言实现，两版输出完全一致。

## 快速开始

### Python 版

```bash
pip install pydantic termcolor
python chan.py
```

### C99 版

```bash
gcc -std=c99 -Wall -Wextra -O2 -o chan chan.c -lm
./chan
```


启用内存调试（编译时无需额外选项，修改 `chan.c` 中的 `CHAN_DEBUG_MEM` 即可）。

## 分析流水线

原始 K 线经过逐层处理得到各级别结构，上层变动时下层自动重建：

```
原始K线 ──> 缠论K线（合并K线） ──> 分型 ──> 笔 ──> 线段 ──> 中枢
                                     │          │            │
                                     └─ 笔中枢 ─┘─ 线段中枢 ─┘─ 扩展线段 / 线段线段 / …
```

`观察者`（Observer）是顶层调度器，持有全部序列数组并驱动流水线。`观察者.增加原始K线()` 每次接收一根新 K 线，触发整个结构的增量重算。核心理念是 **“变动即重建”**——新 K 线导致分型失效时，包含它的笔被弹出，进而可能使线段失效、中枢失效，逐层向上传播。

## 对象模型与内存管理（C 版）

C 版基于**引用计数**实现自动内存管理，所有动态对象通过统一接口创建和销毁：

```c
// 分配对象（引用计数初始为 1）
void* 分配(size_t 大小, 对象类型 类型);

// 增加引用计数（获取所有权）
void 引用(void* 对象);

// 减少引用计数（引用计数归零时自动析构并释放）
void 解引用(void* 对象);
```

所有结构体首字段嵌入 `对象头`（引用计数 + 类型标记），析构时通过类型分派执行对应的 `释放Xxx()` 函数。

### 强引用 vs 弱引用

| 引用类型 | 操作 | 典型场景 |
|----------|------|----------|
| 强引用 | `引用()` / `解引用()` | 动态数组元素、线段→笔、虚线→中枢序列、虚线→线段特征 |
| 弱引用 | 裸指针赋值（不操作引用计数） | 分型→缠论K线、虚线→分型、中枢→虚线元素 |

弱引用不会导致对象悬挂，因为重构顺序保证高层结构总在低层对象被销毁之前清理完毕。

### 回收站与循环释放

程序运行期间，弹出/替换产生的孤立对象统一放进**回收站**。程序退出时 `释放观察者()` 执行五阶段清理：

1. **移除序列引用** — 逐条释放观察者级序列中的所有元素引用
2. **去重 + 移除创建引用** — 对回收站去重，为每个对象增加临时引用保护后移除创建引用
3. **多遍级联释放** — 逐轮释放无外部引用的叶节点，析构级联降低父节点依赖
4. **移除临时引用** — 清理临时保护引用，让引用计数反映真实状态
5. **强制打破循环** — 对纯引用计数无法解决的循环引用（如线段↔线段特征、虚↔中枢），直接断开内部数组并释放

经验证，完整分析 50,003 根 K 线的数据后，全部约 395 万次内存分配无一泄漏（Valgrind 零错误）。

## 数据类型速查

| Python 类 | C 结构体 | 含义 |
|-----------|----------|------|
| `K线` | `K线` | 原始 K 线，含 OHLCV + MACD/RSI/KDJ 指标 |
| `缠论K线` | `缠论K线` | 合并后的缠论 K 线，含方向与分型结构 |
| `分型` | `分型` | 三根相邻缠论 K 线构成的顶/底/上升/下降形态 |
| `虚线` | `虚线` | 通用虚线结构，同时用于笔和线段 |
| `线段特征` | `线段特征` | 构成线段特征序列的笔组 |
| `中枢` | `中枢` | 三段连续笔/线段的重叠区间，含第三买卖点 |
| `观察者` | `观察者` | 顶层调度器，持有全部序列，驱动流水线 |
| `缠论配置` | `缠论配置` | 分析参数与开关 |

## 核心术语

| 术语 | 含义 |
|------|------|
| 普K | 原始 K 线 |
| 缠K | 合并后的缠论 K 线 |
| 笔 | 按缠论规则连接相邻分型形成的线段 |
| 线段 | 由笔序列构成的更高级别段 |
| 中枢 | 三段连续笔/线段重叠形成的价格中枢 |
| 分型 | 三根相邻缠论K线的形态（顶分型/底分型） |
| 文/武 | 起点/终点（civil/martial） |
| 缺口 | 两段之间未重叠的区域 |

## 数据格式

`.nb` 文件为大端序二进制，每条记录 6 个 `double`（48 字节）：

```
时间戳, 开盘价, 最高价, 最低价, 收盘价, 成交量
```

文件名编码元信息：`{品种}-{周期}-{起始时间戳}-{终止时间戳}.nb`
