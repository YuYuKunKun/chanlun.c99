# chanlun.c99 —— 缠论技术分析库

基于**缠论（缠中说禅理论）**的金融 K 线多层次技术分析库，提供 **Python 参考实现**与 **C99 高性能移植**双版本，两版输出**逐笔完全一致**。附带 **Python C API 原生扩展**，将 C 内核封装为 Python 可直接调用的模块，零 FFI 开销。

> 总代码量约 **18,600 行**（C 约 13,200 行，Python 约 5,400 行），含完整测试覆盖。

### 1. 双语言实现 + 原生扩展

```
Python 业务层 (chan_c99/)
    │
    ▼  Python C API（零 FFI 开销）
    │
C 内核 (chan.c / chan.h)  ←──  独立 C 程序可脱离 Python 运行
    │
    ▼
Python 参考实现 (chan.py)  ←──  双版交叉验证，输出完全一致
```

- **C 内核**：`chan.c`（5,934 行）+ `chan.h`（1,002 行），纯 C99 单编译单元，无外部依赖（仅需 libm + libpthread）
- **Python 绑定**：`_core.c`（6,226 行），基于 Python C API 手写，涵盖 17+ PyTypeObject，支持 Python 子类化、自定义 `__repr__`/`__str__` 回调
- **Python 参考**：`chan.py`（3,625 行），完整可运行，作为 C 版的对照基准

### 2. 内存池（Bump-Pointer Arena）

每个 `观察者` 持有独立内存池，整套分析过程中所有对象从池内分配，观察者销毁时整池释放：

- **Bump-Pointer 分配**：~5ns/次，两条指令完成，零系统调用。块内空间不足时自动 2x 扩容分配新块，单次大对象走 Oversized 专用块
- **引用计数保留**：`分配()` 初始化引用计数=1，`引用()`/`解引用()` 用 `__atomic` 操作保证多线程安全。解引用归零时执行 `对象销毁()` 清理内部资源，但**不调 `free()`**——内存归池所有，整池统一释放
- **强弱引用分离**：动态数组元素持有强引用（操作引用计数），分型→K线、虚线→分型等结构关系用裸指针弱引用，由重建顺序保证无悬空
- **清理链表**：池内对象通过侵入式链表串联，释放观察者时遍历清理链表统一销毁，无需 GC 扫描
- **多线程安全**：全局池受 `pthread_mutex_t` 保护，分配/释放用 `__atomic` CAS 无锁操作，`观察者` 支持跨线程并发创建、分析和释放

### 3. 变动即重建（Rebuild-on-Change）架构

分析流水线采用增量更新策略——当新 K 线导致底层结构失效时，失效信号沿依赖链向上传播：

```
原始K线 → 缠论K线（合并） → 分型 → 笔 → 线段 → 中枢
                              │       │       │
                              └── 笔中枢 ──┘── 线段中枢 ──┘── 扩展线段 / 线段线段
```

每个 `观察者` 实例持有 10+ 个序列数组，每次 `增加原始K线()` 调用触发全流水线重算。

### 4. 多周期合成

`K线合成器` 从低周期 K 线实时合成高周期，支持多级并行合成（如 5min → 25min → 150min），`立体分析器` 在多周期上独立分析各自的市场结构。

## 构建与运行

### C 独立程序

```bash
gcc -std=c99 -Wall -Wextra -O2 -finput-charset=UTF-8 -o chan chan.c -lm -lpthread
./chan                      # 从默认 .nb 文件读取并分析
./chan --syncer             # K线合成器一致性验证
```

### Python 原生扩展

```bash
pip install -e .            # 自动检测编译器 UTF-8 支持
python -m pytest tests/test_consistency.py -v   # 51 项一致性测试
```

## 扩展能力

| 能力 | 说明 |
|------|------|
| **Python 子类化** | `观察者`、`笔`、`线段`、`中枢` 可被 Python 继承并覆写方法 |
| **上下文管理器** | `with 观察者(...) as obs:` 自动释放 C 资源 |
| **自定义 repr/str** | `_设置自定义strrepr(类型, ...)` 动态注入 Python 回调 |
| **枚举中文化** | `方向`（向上/向下/…）、`分型结构`（顶/底/…）返回中文枚举成员 |
| **动态数组** | C `动态数组` 的 Python list 风格包装，append/pop/clear/索引 |
| **背驰分析** | MACD / 斜率 / 测度 / 全量 / 任意 / 配置 / 任选 共 7 种背驰判断 |
| **K线合成器** | 低周期 → 高周期合成，多周期并行，回调通知 |

## 核心数据类型

| 类型 | 含义 |
|------|------|
| `K线` | 原始 K 线（OHLCV + MACD/RSI/KDJ 指标） |
| `缠论K线` | 合并后的缠论 K 线，含方向与分型结构 |
| `分型` | 三根相邻缠论 K 线构成的顶/底形态 |
| `笔` | 相邻分型之间的 K 线序列 |
| `线段` | 重叠笔序列构成的高级别段 |
| `中枢` | 三段连续笔/线段的重叠价格区间 |
| `缺口` | 两个价格区间之间的未重叠区域 |
| `观察者` | 顶层调度器，持有全部序列并驱动分析流水线 |
| `缠论配置` | 分析参数配置（~80 个配置项） |

## 数据格式

`.nb` 文件为大端序二进制，每条记录 48 字节（6 × double）：

```
时间戳, 开盘价, 最高价, 最低价, 收盘价, 成交量
```

文件名编码：`{品种}-{周期秒}-{起始时间戳}-{终止时间戳}.nb`
