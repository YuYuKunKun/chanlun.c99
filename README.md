# chan2c99 — 缠论技术分析库（Python 参考实现 + C99 移植）

基于缠论的多层次 K 线技术分析库。提供 **Python** 与 **C99** 双语言实现，两版输出完全一致。

## 快速开始

### Python 版

```bash
pip install pydantic termcolor
python chan.py
```

### C99 版

```bash
gcc -std=c99 -Wall -Wextra -O2 -o chan chan.c -lm
./chan
```

运行后从 `.nb` 数据文件中读取 K 线，逐条增量计算，输出 `c_笔.txt`、`c_线段.txt`、`c_中枢.txt`。

启用内存调试（编译时无需额外选项，修改 `chan.c` 中的 `CHAN_DEBUG_MEM` 即可）。

## 分析流水线

原始 K 线经过逐层处理得到各级别结构，上层变动时下层自动重建：

```
原始K线 ──> 缠论K线（合并K线） ──> 分型 ──> 笔 ──> 线段 ──> 中枢
                                     │          │            │
                                     └─ 笔中枢 ─┘─ 线段中枢 ─┘─ 扩展线段 / 线段线段 / …
```

`观察者`（Observer）是顶层调度器，持有全部序列数组并驱动流水线。`观察者.增加原始K线()` 每次接收一根新 K 线，触发整个结构的增量重算。核心理念是 **“变动即重建”**——新 K 线导致分型失效时，包含它的笔被弹出，进而可能使线段失效、中枢失效，逐层向上传播。

## 对象模型与内存管理（C 版）

C 版基于**引用计数**实现自动内存管理，所有动态对象通过统一接口创建和销毁：

```c
// 分配对象（引用计数初始为 1）
void* 分配(size_t 大小, 对象类型 类型);

// 增加引用计数（获取所有权）
void 引用(void* 对象);

// 减少引用计数（引用计数归零时自动析构并释放）
void 解引用(void* 对象);
```

所有结构体首字段嵌入 `对象头`（引用计数 + 类型标记），析构时通过类型分派执行对应的 `释放Xxx()` 函数。

### 强引用 vs 弱引用

| 引用类型 | 操作 | 典型场景 |
|----------|------|----------|
| 强引用 | `引用()` / `解引用()` | 动态数组元素、线段→笔、虚线→中枢序列、虚线→线段特征 |
| 弱引用 | 裸指针赋值（不操作引用计数） | 分型→缠论K线、虚线→分型、中枢→虚线元素 |

弱引用不会导致对象悬挂，因为重构顺序保证高层结构总在低层对象被销毁之前清理完毕。

### 回收站与循环释放

程序运行期间，弹出/替换产生的孤立对象统一放进**回收站**。程序退出时 `释放观察者()` 执行三阶段清理：

1. **移除序列引用** — 逐条释放观察者级序列中的所有元素引用，幸存者入回收站
2. **去重 + 移除创建引用** — 对回收站去重，用临时引用保护后移除创建引用
3. **打破循环 + 强制释放** — 释放虚线的内部中枢与线段特征，清空所有内部数组以断开引用环，然后移除临时引用并强制释放残余对象

引用环（虚线↔中枢、虚线↔线段特征）的参与者通过直接 `free()` 释放（不依赖引用计数归零），无需多遍级联扫描。进程退出时 OS 回收剩余资源。

经验证，完整分析 50,003 根 K 线的数据后，AddressSanitizer 零报错。

| `虚线` | `虚线` | 通用虚线结构，笔和线段的基类 |
| `线段特征` | `线段特征` | 构成线段特征序列的笔组 |
| `中枢` | `中枢` | 三段连续笔/线段的重叠区间，含第三买卖点 |
| `动态数组` | `动态数组` | Python list 风格包装，支持 append/pop/len/[] |
| `背驰分析` | — | 背驰（背离）分析——斜率/MACD/测度/全量等方法 |
| `观察者` | `观察者` | 顶层调度器，持有全部序列，驱动流水线 |
| `缠论配置` | `缠论配置` | 分析参数与开关，含 ~80 个配置字段 |

## 核心术语

| 术语 | 含义 |
|------|------|
| 普K | 原始 K 线 |
| 缠K | 合并后的缠论 K 线 |
| 笔 | 按缠论规则连接相邻分型形成的线段 |
| 线段 | 由笔序列构成的更高级别段 |
| 中枢 | 三段连续笔/线段重叠形成的价格中枢 |
| 分型 | 三根相邻缠论K线的形态（顶分型/底分型） |
| 文/武 | 起点/终点（civil/martial） |
| 缺口 | 两段之间未重叠的区域 |

## 数据格式

`.nb` 文件为大端序二进制，每条记录 6 个 `double`（48 字节）：

```
时间戳, 开盘价, 最高价, 最低价, 收盘价, 成交量
```

文件名编码元信息：`{品种}-{周期}-{起始时间戳}-{终止时间戳}.nb`

### Python 原生扩展

默认使用 Python C API 原生扩展（零 FFI 开销），备选 ctypes 模式：

```bash
# 构建原生扩展
gcc -std=c99 -Wall -Wextra -O2 -fPIC -finput-charset=UTF-8 \
    -I$(python -c 'import sysconfig; print(sysconfig.get_path("include"))') \
    -c -o _core.o _core.c
gcc -std=c99 -Wall -Wextra -O2 -fPIC -finput-charset=UTF-8 \
    -c -o chan_py.o chan.c
gcc -shared -lm -o _core.so _core.o chan_py.o
ln -sf $PWD/_core.so chan_c99/_core.so

# 运行测试
python -m pytest tests/test_consistency.py -v

# 切换回 ctypes 备选模式：
CHAN_C99_MODE=ctypes python ...
```

**扩展能力：**

- **Python 子类化** — `观察者`、`笔`、`线段`、`中枢` 可被 Python 继承
- **自定义 `__repr__`/`__str__`** — `_设置自定义strrepr(类型, '__repr__', callback)` 向 C 类型注入 Python 回调
- **枚举值中文化** — `方向`（向上/向下/…）和 `分型结构`（顶/底/上/下/散）直接返回中文字符串
- **动态数组** — C 动态数组的 Python list 风格包装，支持混合类型存储
